Part 1: Intro and Review

High level JS

  • Created in 10 days by Brendan Eich in 1995 while working at Netscape
  • Used to create interaction between websites and users
  • JavaScript runs on the client side (aka front end)
  • The programming language of the web
  • Programming is defined as
    A programming language is a formal constructed language designed to communicate instructions to a machine, particularly a computer. Programming languages can be used to create programs to control the behavior of a machine or to express algorithms.
  • Programming language. Tasks are done programaticlly. With logic. Based on this, do that.
  • Common route to JS is through web design and dev. Eventually you have to use it, and it's fun. Makes it different than other programming languages.

HTML, CSS, and...

  • HTML defines the structure of the web page
  • CSS applies the styles
  • JavaScript creates interactions
  • JavaScript has access to the DOM via the DOM API
  • JavaScript knows when the user interacts with the site and can perform actions based on those interactions

High level jQuery

  • jQuery is a open-sourced library
  • Created in 2006
  • Used on over 61% of the top 100,000 websites, over 20 mil overall (2011)
  • Takes a lot of the repetetive tasks that we do in JS and makes them easier
  • Cross browser compatible
  • Talk about open sourced
  • Talk about what a library is
  • Talk about cross browser woes

Functions

  • Great for repeating tasks
  • Can take input values
  • Runs code, then typically returns a value
  • Example of mundane, repetitive task. Now what about if you had a function?

What does a function look like?


function helloWorld() {
	return "Hello World"
};
  • The "funciton" keyword
  • The "return" keyword
  • Nothing after the return keyword, unless in an if / else statement

How to execute a function


helloWorld();
  • Show in console
  • Have class try in console

Now with arguments


function hello(name) {

hello('Noah');

  • Any code inside the function has access to the arguments

Loops

  • Loops iterate through a list (typically an array)
  • Offer a quick and easy way to perform the same task over and over again
  • We're going to focus on the "for" loop

The for loop


for (var i = 0; i < 5; i++) {
	// Run this code 5 times
}
  • Break down each element in the for loop
  • Code won't move on until loop is over

Looping through an array


var array = [10, 20, 30, 40, 50];

for (var i = 0; i < array.length; i++) {
	console.log(array[i])
}
  • Remember how array's are accesed
  • Array[i] will return each item in the array

Running a function in a loop


function add10(number) {
	return number + 10;
};

var array = [10, 20, 30, 40, 50];

for (var i = 0; i < array.length; i++) {
	console.log(add10(array[i]));
}
  • The functions input is each item in the array
  • Run function by itself, then run loop example in console.

  • It does add some "weight" to your page

Selecting elements without jQuery


document.getElementById('elementId');
  • There are many different ways to select an element in plain JS, not all is supported in every browser
  • Once an element is selected, there are a lot of steps you need to take to perform an action on that element

jQuery to the rescue


$('.myElement');
  • Uses css selectors to select an element
  • Works in every browser
  • Can perform any jQuery method on that element very easily

Using jQuery methods


$('.myElement').addClass('active');
  • Uses "dot" syntax
  • There are hundred of methods you can use
  • Show examples using console
  • Whenever you $().[anything], you can just look that up in the docs

Some methods

.addClass()

$('.myElement').addClass('newClass');

.removeClass()

$('.myElement').removeClass('newClass');

.toggleClass()

$('.myElement').toggleClass('newClass');
  • Code sample here

The jQuery docs

https://api.jquery.com

  • A complete list of methods

The click event


$('.myElement').click(function() {
	// When .myElement is clicked, this code will run
});
  • This is called an event listener
  • This code will run once the .myElement element is clicked on

The "this" keyword


$('.myElement').click(function() {
	// $(this) will select the element you just clicked on

	$(this).addClass('clicked');
});
  • The "this" keyword refers to the element you just clicked on
  • It has a very special meaning in JS
  • Use the same jQuery selector syntax, but you dont need quotes
  • Useful when you are selecting many elements at once, but only want to manipulate the element that was just interacted with
  • Can use with any event
  • Talk about a list of elements, change only the element you clicked
  • You can select each one individually, or use the this keyword and only select once
  • Code sample here: show difference between just using the same selector again vs using this inside a click event

Part 5: jQuery 2

Event Driven Programming

  • JS is an event driven programming language
  • jQuery gives us nice shortcuts to "bind" events
  • You may see the .on() method instead
  • Don't forget to use .off()!
  • Code Sample here - 2 .on() methods, then show .off()

More methods

.siblings()

$('.myElement').siblings();

// or include a selector
$('.myElement').siblings('.selector');

.attr()

// get an attribute's value
$('.myElement').attr('class');

// set an attribute's value
$('.myElement').attr('class', 'newClass');

.css()

// get a css style
$('.myElement').css('background-color');

// set a css value
$('.myElement').css('background-color', '#fff');
  • attr get, set pattern a common one
  • Notice in sibling you need to include the dot, but not in the attr set method
  • There are ways to access many different properties and change different properties with one call. Go to docs to find out.

Chaining jQuery methods


$('.myElement').click(function() {
	// $(this) will select the element you just clicked on

	$(this).addClass('clicked').fadeOut().fadeIn().siblings().fadeOut().fadeIn();
});
  • You can chain together as many methods as you like
  • Don't forget the parenthesis each time
  • Be careful if you change which element you are targeting

Seperation of conerns

  • Time's have changed with CSS3
  • Leave all your style definitions in CSS
  • Even your animations now can go in your CSS
  • Manipulating CSS classes is very powerful
  • Use jQuery to target elements on events
  • The difference between using jQuery 5 years ago, and now. (Animations)
  • jQuery should be used for events, targeting elements, AJAX, but NOT STYLES

The power of classes and jQuery together

Just jQuery


$('.myElement').css('background-color', #336699);

$('.anotherElement').css('background-color', #336699);

$('.someText').css('color', #336699);

Now with CSS


// JavaScript
$('body').addClass('blue');

// CSS
body.blue .myElement {
	background-color: #336699;
}
body.blue .anotherElement {
	background-color: #336699;
}
body.blue .someText {
	color: #336699;
}
  • Maybe not less lines of code, but way more readable
  • Consider other people looking at your code, or if you were to come back to it months later

More methods

.hide() and .show()

$('.myElement').hide();

$('.myElement').show();

.fadeOut() and .fadeIn()


$('.myElement').fadeOut();

// can pass it a speed
$('.myElement').fadeIn(600);
  • what is an alternative way to do hide and show?

Callback functions

$('.myElement').fadeOut(600, function() {
	$('.newElement').fadeIn(600);
});
  • Check docs to see order of arguments
  • The "this" keyword changes value in callbacks
  • Can have as many callbacks as you want, but be careful of readability

AJAX

  • Is not a new programming language
  • Just another jQuery method
  • Loads content from another source onto your page
  • Used to create SPAs
  • http://api.jquery.com/jquery.ajax/

$.ajax({
	url: , // url of where the content is coming from
	data: , // data to be passed in the request
	success: function() { // Code to run after you get the content from the request },
	error: function() { // Code to run if there was an error on the request }
});
